Part 1: Masses and Springs

1.1 Wireframe Structure

The cloth is modeled as a grid of point masses connected by three types of springs:

  • Structural springs: Connect adjacent point masses horizontally and vertically, providing stiffness
  • Shearing springs: Connect diagonal neighbors, providing resistance to shearing forces
  • Bending springs: Connect point masses two steps apart, providing resistance to bending

Part 2: Simulation via Numerical Integration

2.1 Default Configuration

The simulation uses Verlet integration to update point mass positions based on forces from springs and external accelerations (gravity). Each simulation step computes new positions using the current and previous positions.

2.2 Effect of Spring Constant (ks)

The spring constant ks controls how strongly springs resist stretching. Higher ks means stiffer cloth, while lower ks creates more elastic, flowing behavior.

Observations:

  • Low ks (10-100): The cloth is very elastic and floppy. It stretches significantly under gravity and takes longer to settle. The folds are deep and dramatic.
  • High ks (10000-50000): The cloth is very stiff and resists stretching. It falls more like a rigid body and settles quickly with minimal deformation.

2.3 Effect of Density

Density affects the mass of each point, which influences how gravity affects the cloth. Higher density means heavier cloth that falls faster and stretches more under its own weight.

2.4 Effect of Damping

Damping controls energy loss during simulation. Higher damping causes faster settling but can make the cloth feel sluggish.

Part 3: Handling Collisions with Other Objects

3.1 Collision Detection Implementation

Sphere Collision: For each point mass, I check if its distance to the sphere's center is less than the sphere radius. If collision is detected, I compute the tangent point on the sphere surface and apply a correction vector scaled by the friction coefficient.

Plane Collision: For each point mass, I check if it crosses the plane by comparing signed distances of the last and current positions. If a crossing is detected, I interpolate to find the intersection point and offset slightly above the surface.

3.2 Sphere Collision Results with Different ks

  • ks=50000: The cloth is very stiff and doesn't wrap as much. It sits more on top of the sphere with minimal deformation.

4.1 Implementation Using Spatial Hashing

Self-collision detection uses spatial hashing for O(n) average complexity. The space is partitioned into uniform boxes, and each point mass is mapped to a bucket based on its position. For collision response, each point checks neighbors in the same and adjacent buckets.

Part 5: Shaders

5.1 Shader Program Basics

A shader program is a set of programs that run on the GPU to control the graphics rendering pipeline. The vertex shader processes each vertex individually, transforming positions and computing per-vertex attributes like normals and texture coordinates. The fragment shader then processes each pixel, computing the final color based on interpolated vertex data, textures, and lighting calculations.

5.2 Blinn-Phong Shading Model

The Blinn-Phong model combines three lighting components:

  • Ambient: A constant color representing indirect light, ensuring objects aren't completely dark
  • Diffuse: Light scattered uniformly in all directions, based on the angle between the surface normal and light direction
  • Specular: Bright highlights that appear on shiny surfaces, computed using the halfway vector between light and view directions

The final color is: color = ambient + diffuse * (N·L) + specular * (N·H)^p

5.3 Texture Mapping

Texture mapping applies a 2D image to the cloth surface using UV coordinates. The fragment shader samples the texture at each pixel's UV position to get the base color.

5.4 Bump Mapping vs Displacement Mapping

Bump Mapping: Modifies only the surface normals to create the illusion of bumps and grooves. The geometry remains unchanged, but lighting creates the appearance of depth.

Displacement Mapping: Actually moves the vertices along the normal direction based on the height map, changing the true geometry of the mesh.

Displacement Mapping with Different Mesh Resolutions

Comparison:

  • Coarse mesh (16x16): Bump mapping looks better than displacement because it doesn't have enough vertices to represent the displacement properly. Displacement creates visible facets and edges.
  • Fine mesh (128x128): Displacement mapping looks significantly better with smooth, detailed geometry. Bump mapping still works but the mesh has enough vertices to capture the displacement.
  • Key difference: Bump mapping preserves smooth silhouettes while displacement actually changes the shape. Bump is faster but displacement is more physically accurate.

5.5 Mirror Shader

The mirror shader uses a cubemap texture to create reflections. The view direction is reflected off the surface normal and used to sample the environment cubemap.

5.6 Custom Shader

My custom shader combines all 5 shader techniques with procedural animations:

  • Vertex Shader: Displacement mapping with procedural wave animation
  • Fragment Shader:
    • Texture mapping: Blends texture_1 and texture_2 with time-based animation
